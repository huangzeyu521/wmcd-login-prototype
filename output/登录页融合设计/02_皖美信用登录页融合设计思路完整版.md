## 模块一 · 设计核心定位与融合原则

### 1.1 一句话设计主张
> **"启信为钤、钤动成美"——以安徽国资"皖美信用之印"为主角，让每一次登录都成为一次带仪式感的正向信用启封。**

### 1.2 定位三元锚
| 锚点 | 内容 |
|---|---|
| 品牌锚 | 安徽国资 · 皖美信用 · 省级权威 · 监管合规 |
| 情感锚 | 稳重而非刻板、庄严而有温度、有巧思而不花哨 |
| 功能锚 | 3 秒内可见登录主路径、5 秒内可完成标准登录、0 秒障碍 |

### 1.3 融合原则（FIVE-PACT 五约）
| 编号 | 原则 | 约束 |
|---|---|---|
| PACT-1 | **仅承继哲学，不承继形骸** | 只复用"四段闭环+五必要条件+三件套反馈"哲学，禁用任何台灯具象 |
| PACT-2 | **国资为体，巧思为用** | 巧思服务于国资品牌表达，巧思绝不凌驾于权威性之上 |
| PACT-3 | **强隐喻一换到底** | 把"开灯照亮"整体替换为"启封钤印"，不做"半换"（例如保留灯+换灯罩材质是被禁止的） |
| PACT-4 | **色彩封域** | 色域严格封闭在皖美红金五色体系内，禁用彩虹漂移 |
| PACT-5 | **功能优先级永高于动效** | 动效、音效、光影任何一项与登录核心功能冲突时，先砍动效 |

### 1.4 如何平衡"专业性 × 趣味性"
- **走"仪式感"而非"游戏感"**：仪式感（庄严的开启动作）是国资场景允许的情绪增量；游戏感（萌系/弹性/随机性）是禁区。
- **走"可跳过的巧思"而非"必须玩的关卡"**：默认提供"直达登录"的快速通道（键盘 Tab/无障碍链接），不让趣味交互成为可用性门槛。
- **走"一次性惊喜"而非"反复打扰"**：趣味交互仅出现在首次访问的初始状态，已登录过的用户 sessionStorage 记忆后直接呈现表单。

### 1.5 如何规避照搬抄袭
- 具象元素完全不同：**皖美信用之印 ≠ 台灯**
- 主题隐喻完全不同：**启封钤印 ≠ 开灯照亮**
- 色彩体系完全不同：**红金五色 ≠ 彩虹漂移**
- 动画哲学完全不同：**中式稳润 ≠ 西式弹性**
- 音效语义完全不同：**绶带落印 ≠ 拉绳咔哒**
- 版权署名完全不同：**安徽国资监制 ≠ Lumina Studio 版权**
- 仅保留："用户主动 → 双状态 → 功能区揭示 → 全页面联动"这 1 条底层哲学

## 模块二 · 核心交互元素与完整交互闭环设计

### 2.1 核心交互元素：皖美信用之印（原创重构）

#### 2.1.1 元素定义
```
名   称：皖美信用之印（简称"信印"）
形   制：方形印章 × 徽派檐角压印
印   面：刻四字"皖美信用"（篆隶融合体）
印   钮：徽派马头墙抽象化钮顶（非写实、非卡通，极简几何）
印   盒：朱漆绶带 + 徽墨底座 + 淡金描边
站   位：登录页左侧视觉中心（与右侧功能区对称均衡）
原创声明：本元素由本次设计原创组合，印章语义取自中国官方权威符号体系，
         徽派檐角取自安徽本土文化，"皖美信用"四字为本平台注册品牌词
```

#### 2.1.2 为什么选"印"不选台灯
| 维度 | 印章的适配性 |
|---|---|
| 国资合规 | 印章是中国官方权威最高级符号，100% 匹配国资监管调性 |
| 信用隐喻 | 印章天然绑定"契约、担保、落定"三重信用语义 |
| 双状态天然 | "封印—启印"本身就是完美的二元状态，映射"未登录—已登录" |
| 地域识别 | 徽派元素嫁接后具备不可替代的"安徽"属性 |
| 禁娱乐化 | 印章气质天然克制，与娱乐化绝缘 |

#### 2.1.3 元素的双状态定义
| 状态 | 命名 | 视觉形态 | 语义 |
|---|---|---|---|
| A · 封存态（默认） | **"信印封存"** | 信印静卧于徽墨印盒中，朱红绶带系扣未启，印面朝下蒙尘，印钮微暗 | 信用未启，登录未动 |
| B · 启封态（激活） | **"信印已启"** | 信印升起悬浮于印盒上方约 20px，绶带飘落，印面翻转朝外，朱泥沾润，落下一枚 1:1 真实印鉴印记 | 信用已启封，登录已就绪 |

### 2.2 主题化交互闭环（完整用户路径）

#### 2.2.1 全流程路径图
```
[段0·进场]  页面加载 → 徽墨渐显 → 信印居左呈封存态 → 右侧呈"待启封"占位
                                │
                                ▼
[段1·邀约]  0.8s 后，印盒绶带系扣处出现极弱的"一次性金色呼吸光"
            配合微文字提示："拨动绶带以启封"（仅 2 秒显现后淡出）
                                │
                                ▼
[段2·操作]  用户鼠标/指针移至绶带系扣 → 光标变为"抓取态"（国密锁形光标 fallback 常规 grab）
            用户向下拨动绶带（可拖拽，支持键盘 Enter 等价触发）
            过程中：
              · 绶带随拖拽幅度平滑延展，呈现真实布面丝缎物理感
              · 印盒同步微抖（幅度 ≤ 2px），表达"即将启封"的临界感
              · 伴随极轻绶带丝声（低频白噪，时长 ≤ 120ms）
                                │
                                ▼
[段3·判定]  拖拽位移 ≥ 阈值 T（相对印盒高度的 40%）时判定为有效启封
            未达阈值释放：绶带以"中式阻尼回弹"归位（三段式：缓出 → 小过冲 → 稳定）
            达到阈值释放：进入段4
                                │
                                ▼
[段4·落印]  信印升起（0.4s 缓入）→ 印面翻转（0.2s 逆时针 180°）→
            朱泥润印（0.15s 色相自徽墨渐变为朱红）→
            落印（0.25s 缓压至印盒上方，配"嗡"的一次性低频落印音，响度 ≤ 6dB）→
            在页面中心下方浮现一枚"皖美信用"真实印鉴（0.3s 由浅至深，0.5s 保持，渐隐融入底纹）
                                │
                                ▼
[段5·揭示]  右侧登录表单自"卷轴展开"方式呈现（卷轴宽度由 0 → 100%，0.5s 缓出）
            表单各字段从上至下错峰进场（每字段 80ms stagger）
                                │
                                ▼
[段6·照映]  全页面光晕呼吸开启：
            · 背景徽墨 → 徽墨带极弱朱金呼吸光（呼吸周期 8s，幅度 ≤ 6%）
            · 顶部安徽国资 LOGO 保持常亮不变（品牌稳定性约束）
            · 底部"皖美信用"水印由 0% → 4% 透明度渐显
                                │
                                ▼
[回退]     用户可点击表单右上角"归印"按钮（极小图标），触发反向闭环：
            表单卷轴回卷 → 印鉴淡出 → 信印回落封存 → 绶带系扣重现 → 背景呼吸光熄灭
            该操作为非必要功能，仅供误触保护与演示场景
```

#### 2.2.2 每一步的操作反馈规则
| 步骤 | 视觉 | 动效 | 音效 | 触觉（未来预留） |
|---|---|---|---|---|
| 段1 邀约 | 金色呼吸光 2s 淡入淡出 | 无 | 无 | 无 |
| 段2 操作过程 | 绶带随拖拽延展，印盒微抖 | 连续跟随，1:1 位移映射 | 丝缎低频白噪 ≤ 120ms | 可选振动 |
| 段3 回弹 | 绶带三段阻尼回弹 | 中式阻尼曲线 | 无 | 无 |
| 段4 启封 | 印章升起+翻转+润印+落印四段 | 严格"起承顿收"四拍节奏 | 一次性低频"嗡"落印音 ≤ 6dB | 一次短振 |
| 段5 揭示 | 卷轴展开 + 字段 stagger | 缓出 + 错峰 | 极轻纸展声 ≤ 80ms | 无 |
| 段6 照映 | 全页面呼吸光 | 正弦呼吸 8s | 无 | 无 |
| 回退 归印 | 全部反向动画 | 反向曲线 | 极轻收印声 ≤ 80ms | 无 |

#### 2.2.3 判定条件与可控性底线
- 拖拽阈值 T 需在用户研究后由数据标定，默认 40%。
- 若用户 0.5s 内连续触发 3 次未达阈值释放，系统自动弹出"改用常规登录"的辅助引导，保护弱操作能力人群。
- 整个交互 2s 无响应即判定为"环境异常"，自动降级为传统静态登录页。

### 2.3 三条可替代主路径（权威性 > 趣味性）
| 通道 | 触发条件 | 路径 |
|---|---|---|
| **主通道·仪式启封** | 首访/默认/有鼠标用户 | 上文段0–段6 |
| **快捷通道·键盘启封** | Tab 聚焦到信印时按 Enter/Space | 跳过段2过程动画，直接走段4–段6 |
| **无障碍通道·直接登录** | 屏幕阅读器用户/偏好降级用户/sessionStorage 已登录过 | 跳过段0–段5，直接呈现静态表单 + 登录按钮获得焦点 |

## 模块三 · 视觉与动画设计规范（双版本）

### 3.1 色彩封域 · 皖美红金五色体系（跨版本共享）

| 色号 | 色名 | 色彩角色 | 典型用途 |
|---|---|---|---|
| WM-01 | **徽墨玄** | 沉稳底色 | 深色版背景、文字 |
| WM-02 | **皖白宣** | 克制浅底 | 浅色版背景、表单底 |
| WM-03 | **朱泥红** | 核心主色 | 信印印面、主 CTA、合规强调 |
| WM-04 | **鎏金晖** | 权威高光 | 边框描金、启封光晕、LOGO 辅色 |
| WM-05 | **黛青线** | 辅助线条 | 分隔线、装饰徽纹 |

**封域规则**：页面中任何动态变色必须在 WM-03/WM-04 之间的狭窄色域内微调，**禁止溢出**。

### 3.2 版本一：**深色沉浸专业版**（会议室/大屏展示）

#### 3.2.1 配色基调
- 背景主色：徽墨玄（WM-01）为底
- 装饰暗纹：徽派马头墙 + 淡水远山 SVG 暗纹，透明度 ≤ 4%
- 主交互色：朱泥红（WM-03）
- 光晕高光：鎏金晖（WM-04），仅出现在"启封后"
- 整体对比度 ≥ 7:1，满足大屏远观

#### 3.2.2 布局规范
- 主栅格：12 列，中线居中
- 信印居左 5 列，登录面板居右 5 列，两侧各 1 列留白
- 顶部 80px 锚定：左 = 安徽国资委徽章 + 皖美信用文字 LOGO，右 = 政务网 SSO + CA 登录 + 无障碍 + 语言切换
- 底部 64px 锚定：合规提示 + 版权 + 备案号 + 主管单位署名

#### 3.2.3 动画节奏（中式稳润八字诀：起 · 承 · 顿 · 收）
| 节奏阶段 | 曲线 | 持续 |
|---|---|---|
| 起（进入） | easeOutCubic | 0.3–0.4s |
| 承（延展） | easeInOutQuart | 0.2–0.3s |
| 顿（到位） | steps(1) 或 back.out 极弱过冲 | 0.05–0.1s |
| 收（稳定） | easeOutSine | 0.15s |

**严禁**使用 `elastic.out(1, 0.4)` 这类西式强弹性曲线。

#### 3.2.4 光影联动逻辑
- 全局光晕仅在段6启用，且呼吸幅度 ≤ 6%，周期 8s，避免注意力竞争
- 印章落印瞬间产生一次"朱金扩散波"，半径 ≤ 240px，透明度 0.3 → 0，时长 0.6s，仅出现一次

### 3.3 版本二：**浅色稳重办公版**（日常办公电脑）

#### 3.3.1 配色基调
- 背景主色：皖白宣（WM-02）
- 装饰暗纹：宣纸纤维 + 徽派线描远山，透明度 ≤ 8%
- 主交互色：朱泥红（WM-03），饱和度较深色版降低 10% 以适配浅底
- 光晕高光：鎏金晖（WM-04）转为"淡金晕"，透明度上限 12%
- 整体对比度 ≥ 4.5:1（WCAG AA）

#### 3.3.2 关键差异
| 维度 | 深色版 | 浅色版 |
|---|---|---|
| 光影手段 | 光晕 + 辉光 | 质感 + 阴影 |
| 印章材质 | 金边高光印章 | 纸贴印章（轻阴影 + 纤维纹） |
| 启封特效 | 朱金扩散波 | 朱色墨晕（类宣纸吸墨） |
| 背景呼吸 | 徽墨 + 金呼吸 | 宣纸 + 暖白呼吸 |
| 音效音量 | ≤ 6dB | ≤ 4dB（办公场景更安静） |

#### 3.3.3 版本切换逻辑
- 用户偏好：系统级 prefers-color-scheme 检测 + 页面右上角"皖墨/皖白"切换键
- 会议室大屏：通过 URL 参数 `?mode=dark` 固定深色版
- 企业部署：支持管理员在后台配置默认版本

### 3.4 CSS 设计令牌绑定规则（抽象描述，无代码）
- 所有状态相关的色值、透明度、阴影半径、模糊半径都抽象为设计令牌
- 令牌在双版本之间通过同名映射切换
- "封存—启封"状态切换通过**一个根级状态属性**驱动，所有组件订阅该属性
- 令牌的数值范围严格封域，禁止运行时随机生成

### 3.5 动画与交互状态的强绑定规则
| 状态 | 绑定动画 | 是否可中断 |
|---|---|---|
| 封存态静置 | 印盒极弱呼吸（幅度 ≤ 1px，周期 12s） | 可 |
| 绶带拖拽中 | 1:1 位移跟随 | 不可（优先级最高） |
| 释放未达阈值 | 三段阻尼回弹 | 不可 |
| 启封四拍 | 起承顿收 | 不可（保护仪式感） |
| 卷轴展开 | 宽度 + 字段 stagger | 可（用户提前按 Enter 立即完成） |
| 照映呼吸 | 正弦 8s | 可（任何交互暂停呼吸） |

## 模块四 · 全量功能融入设计（100% 覆盖、易用性零损）

### 4.1 基础登录功能

#### 4.1.1 账号输入框
- **呈现**：位于表单第一行，宽度占表单 100%
- **样式**：弱化边框 + 下方 1px 黛青线 + 左侧国密账号图标
- **交互**：聚焦时下方线条由黛青变为朱泥红，宽度由短延伸至全宽（0.3s）
- **校验**：失焦即时校验，账号格式错误以红色微线条提示，错误文案位于字段下方 4px 处
- **占位文案**："请输入账号 / 手机号 / 工号"

#### 4.1.2 密码输入框
- **呈现**：位于账号输入框下方 16px
- **样式**：同账号框 + 右侧"显隐切换"图标（国密锁/眼形改为"封/启"双态徽纹图标以贴主题）
- **交互**：同账号框 + 大小写锁定检测时右侧出现"A↑"提示
- **增强**：接入国密 SM2/SM3/SM4 加密，前端仅传密文，账号与密码分别加盐

#### 4.1.3 登录按钮（主 CTA）
- **呈现**：表单最底部，宽度 100%，高度 48px
- **样式**：朱泥红填充 + 鎏金描边 1px + 文字"核验并登录"（白色）
- **悬停**：朱泥红加深 4%，鎏金描边增粗至 2px
- **按下**：0.1s 内下凹 2px，触发一次"钤印"动画——在按钮中心浮现一枚小朱印并渐隐（0.3s）
- **加载**：按钮文字变为"核验中…"，左侧出现小尺寸徽纹旋转加载器（不使用通用 spinner）
- **成功**：按钮变为鎏金底 + 徽墨字"核验通过"，0.6s 后页面进入主平台

#### 4.1.4 记住密码选项
- **呈现**：位于表单底部登录按钮上方 12px，左对齐
- **样式**：**"钤记确认"式自定义勾选框**——未选为空白小方印框，已选为一枚微型朱色小印
- **文案**："记住此账号（适用于本机）"
- **重要**：默认不勾选，符合政务合规（不得默认保留凭据）
- **持久化**：仅记住账号（LocalStorage），密码绝不存本地

#### 4.1.5 忘记密码链接
- **呈现**：位于记住密码同行右端
- **样式**：黛青色文字 + 悬停变朱泥红
- **文案**："忘记密码？请联系所在单位管理员重置"
- **点击**：弹出浮层，引导用户走"单位管理员—组织架构—申诉重置"合规流程（**国资合规不允许自助邮箱重置**）

### 4.2 国资专属功能

#### 4.2.1 安徽国资品牌 LOGO 展示
- **位置**：页面顶部 80px 锚定区左侧
- **组成**：安徽省国资委徽章（16 × 16 小尺寸）+ "皖美信用"中文标识（徽派宋体变体）+ 英文小标识 "Wan-Mei Credit"
- **行为**：LOGO 不参与任何动画，永远保持稳定，作为品牌权威锚点
- **可点击**：点击 LOGO 返回公共门户首页（带确认浮层，防误触）

#### 4.2.2 政务统一身份认证入口（SSO）
- **位置**：页面右上角 + 登录表单下方（双入口，保证可见性）
- **右上角**：文字链接"政务网统一身份认证"+ 微型政务徽标
- **表单下方**：一块独立的"官方认证通道"区块，包含三个并列按钮：
  - 🏛️ 政务网 SSO 登录
  - 🔐 CA 证书登录
  - 📱 国密 U-Key 登录
- **视觉**：三按钮同形不同色，每个按钮左上角有微型权威徽纹，避免与朱泥红主 CTA 色彩混淆

#### 4.2.3 CA 证书登录
- **入口**：官方认证通道区块中部按钮
- **点击流程**：
  1. 检测本机 CA 驱动（5s 超时）
  2. 弹出 CA 密钥输入浮层（政务标准样式，不走常规 modal 风格）
  3. 成功后直接进入平台，跳过二次密码校验
- **降级**：未检测到 CA 驱动时，展示"如何安装 CA 驱动"的帮助卡片，不直接报错

#### 4.2.4 合规提示文案
- **位置**：页面底部 64px 锚定区，分两行
- **第一行**：
  > 本系统受《中华人民共和国数据安全法》《网络安全法》《个人信息保护法》及
  > 《国有企业数据资产监督管理办法》保护。所有访问将被审计留痕，严禁非法访问。
- **第二行**：
  > 本系统已通过等保三级认证 · ISO 27001 认证 · 密码应用安全性评估
- **样式**：黛青色 12px 文字，不可忽略但不喧宾夺主

#### 4.2.5 版权与署名信息
- **位置**：页面最底部
- **内容**：

### 4.3 无障碍与老用户友好设计

#### 4.3.1 无障碍路径
| 障碍类型 | 解决方案 |
|---|---|
| 视力障碍 | 全部图形配 aria-label，信印交互提供 "Enter 键启封" 等价路径 |
| 运动障碍 | 趣味交互可跳过，Tab 键序优先到达表单而非信印 |
| 色弱 | 朱泥红 + 鎏金双色搭配通过色盲模拟测试，不依赖颜色传达关键信息 |
| 认知障碍 | 文案避免生僻字，"钤记确认"等自定义词汇配注解 |
| 老年用户 | 提供"极简登录模式"入口，关闭所有动画，回归静态表单 |

#### 4.3.2 老国资用户习惯保障
- **"跳过仪式"链接**：封存态右上角提供极小号"直接登录"链接，点击跳过全部动画
- **sessionStorage 记忆**：用户首次完整走完启封流程后，同一会话再次进入登录页直接呈现表单
- **高对比度模式**：支持系统级高对比度切换，动画自动降级

### 4.4 登录后的首屏衔接
- 登录成功不使用硬跳转，而以"卷轴继续展开"的方式过渡到平台首页
- 顶部安徽国资 LOGO 位置在首页保持不变，形成连续品牌感知
- 过渡时长 ≤ 0.8s，不让仪式感变成等待

## 模块五 · 原创性差异化说明

### 5.1 与参考样本的全量差异对照
| 维度 | 参考样本（lumina-login） | 本方案（皖美信用登录页） | 差异评级 |
|---|---|---|---|
| 核心交互元素 | 台灯 + 拉绳 | 皖美信用之印 + 朱漆绶带 | ★★★★★ 完全不同 |
| 主题隐喻 | 拉绳 → 开灯 → 照亮表单 | 拨绶 → 启印 → 卷轴展开表单 | ★★★★★ 完全不同 |
| 色彩体系 | 随机 hue 0–360 彩虹漂移 | 红金五色封闭色域 | ★★★★★ 完全不同 |
| 动画哲学 | 西式弹性回弹 elastic.out | 中式"起承顿收"四拍节奏 | ★★★★★ 完全不同 |
| 音效设计 | 拉绳沙沙 + 咔哒金属音 | 绶带丝声 + 落印嗡音 | ★★★★★ 完全不同 |
| 装饰元素 | 抽象彩光光斑 | 徽派马头墙 + 远山 + 宣纸纤维 | ★★★★★ 完全不同 |
| 版权署名 | Lumina Studio | 安徽省国资委主管 + 皖 ICP 备 | ★★★★★ 完全不同 |
| 占位文案 | "拉动灯绳以开启" | "拨动绶带以启封" | ★★★★★ 完全不同 |
| 萌系表达 | 台灯眼睛 + 瞳孔反光 | 无（严禁） | ★★★★★ 完全不同 |
| 双状态语义 | 关灯/开灯（日常生活） | 封存/启封（官方权威） | ★★★★★ 完全不同 |
| 色相随机机制 | 每次开灯随机 hue | 禁用，色域严格封域 | ★★★★★ 完全不同 |

### 5.2 唯一复用点：一条哲学
> "用户主动可控交互 → 触发二元状态切换 → 核心功能区联动揭示 → 全页面视觉呼应"

此条哲学属于通用交互设计方法论，不构成任何著作权/具象设计的复用。类比：所有"下拉刷新"页面共享"下拉→释放→刷新"的交互哲学，但各产品的具象设计千差万别。

### 5.3 原创性声明
- 皖美信用之印形制为本次设计原创组合（印章 × 徽派檐角 × 朱漆绶带 × 篆隶"皖美信用"四字）
- 色彩体系 WM-01 至 WM-05 为本次设计命名原创
- "信印封存 / 信印已启 / 钤记确认 / 核验并登录 / 归印"命名体系为本次设计原创
- "起承顿收"四拍动画节奏为本次设计独立提炼的动画哲学，不承接任何样本动画曲线
- 全部版权归安徽省国资委及承建单位所有

| 显示主管单位与承建单位 | 底部署名区强制呈现三类单位完整信息 |

### 6.2 政务数字化产品合规
| 合规项 | 本方案措施 |
|---|---|
| 《数据安全法》《网络安全法》《个人信息保护法》 | 底部合规提示文案强制告知 |
| 《国有企业数据资产监督管理办法》 | 文案明示 + 后端审计留痕 |
| 等保三级 | 登录流程走 HTTPS + 国密 SM2/SM3/SM4 + 访问行为全量审计 |
| 不得明文传输密码 | 前端国密加密后传输，密文不入本地存储 |

### 8.2 技术选型建议（仅设计层建议，最终以工程团队评估为准）
| 层 | 建议 | 理由 |
|---|---|---|
| 视图层 | Vue 3 / React 18 均可 | 与平台主应用保持一致 |
| 动画引擎 | 优先 CSS + Web Animations API；复杂部分用 GSAP 或 motion | 避免过度依赖，大部分动画原生即可 |
| SVG 交互 | 原生 SVG + DOM 事件；拖拽可用 Pointer Events 或成熟拖拽库 | 国密加密要求资源可审计 |
| 音频 | Web Audio API 合成本地音效，资源体积 0 KB | 同 lumina 技术思路但音色完全不同，不构成复用 |
| 状态机 | XState 或组件内有限状态机皆可 | 封存/启封需严格状态机保证原子切换 |
| 国密 | sm-crypto 或国资自研国密库 | 必须走国密算法 |
| 无障碍 | axe-core 全量扫描 + 人工测试 | 政务合规硬性要求 |
