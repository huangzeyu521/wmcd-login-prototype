# 缺口补全解决方案 + 分步骤执行开发方案

> 覆盖：5 项 P0 + 4 项 P1 缺口
> 原则：最小化侵入 · 保留现有接口 · 复用组件 · 不破坏已实现功能

---

## 一、技术选型（与现有项目一致）

| 层 | 技术 | 理由 |
|---|---|---|
| 框架 | 原生 HTML5 + ES2017+ | 与平台其余页面一致，零打包 |
| UI | Tailwind CDN + shared.css / mvp.css | 复用设计系统 |
| 图表 | **ECharts 5.4 CDN** | 设计文档 11.2.5 指定 |
| 认证 | 复用 WM_AUTH | 不破坏守卫 |
| 组件 | 复用 components.js（renderAuthTopbar）| 顶部导航一致 |
| 数据 | 复用 mock-data.js WM_USERS/WM_STORE | 24 角色 + 企业 |

## 二、缺口补全解决方案

### GAP-P0-01 · Multi-Agent 工作流独立页
- 文件：`frontend/pages/agent-workflow.html`
- 布局（11.3.4）：左 DAG 可视化 + 中执行日志 + 右报告预览 + 顶进度 + 底人工中断
- 技术：SVG 画 DAG 节点 + ECharts graph · 流式 Markdown 渲染 · 节点状态色灯
- 数据：mock-data 新增工作流 demo（投资尽调 DAG）
- 入口：顶部导航「AI 工作流」新增链接 + index 首页入口

### GAP-P0-02 · 个人中心
- 文件：`frontend/pages/profile.html`
- 模块：我的信息卡片 · 角色切换 · 安全设置（密码/CA 绑定）· 操作日志 · 偏好设置入口
- 复用：WM_AUTH.currentUser() · WM_AUTH.switchRole()

### GAP-P0-03 · 系统设置
- 文件：`frontend/pages/settings.html`
- 模块：外观（夜/日）· 字号（小/中/大）· 语言（中简/中繁/English 占位）· 通知偏好 · 隐私授权 · 数据导出
- 实现：localStorage 保存偏好 + 文档 root 属性驱动

### GAP-P0-04 · 帮助中心
- 文件：`frontend/pages/help.html`
- 模块：新手引导（图文步骤）· FAQ（信用/融资/投资/风控分类）· 合规文档（三法一规条文引用）· 联系管理员 · 问题反馈表单
- 数据：静态 FAQ 库

### GAP-P0-05 · 信用异议申请
- 文件：`agent-credit.html` 内新增「异议申请」面板 + 浮层
- 模块：企业录入异议理由 · 上传证据 · 提交 · 状态追踪（待受理/复核中/已调整/驳回）
- 数据：localStorage 存异议列表

### GAP-P1-01 · ECharts 升级
- 引入：index.html、dashboard.html、agent-credit.html 增加 echarts CDN
- 场景：dashboard 地图/趋势柱 · credit 雷达/趋势折线
- 降级：CDN 失败时降级为原 CSS 视图

### GAP-P1-02 · AI 独立对话页
- 文件：`frontend/pages/ai-chat.html`
- 模块：全屏会话 + 历史列表 + 快捷指令
- 复用：ai-client.js 现有能力

### GAP-P1-03 · 快捷键面板
- 实现：components.js 新增 showHotkeysPanel()
- 触发：`?` 键或顶部导航点击
- 列表：Ctrl+K 搜索 · Alt+数字切换智能体 · G+A/F/I/M/S/R 跳转

### GAP-P1-04 · 新手引导 Tour
- 实现：components.js 新增 startFirstRunTour()
- 触发：localStorage 无 `wm_tour_done`
- 步骤：6 步浮层引导（首页→工作台→信用→融资→AI 助手→完成）

## 四、代码规范

- 文件头：与现有页面相同的 script 引用顺序（shared/interactions/mock-data/auth/tickets/phase2/components/ai-client）
- 守卫：`<script>if (!window.WM_AUTH.guard()) throw new Error('not logged in');</script>`
- 顶栏：`document.getElementById('topbar-root').outerHTML = WM_C.renderAuthTopbar('页面名');`
- 命名：CSS 类 `wm-*` · JS 函数 `camelCase` · localStorage key `wm_*`
- 注释：JSDoc 关键函数 + 业务注释说明合规要求
- 版本：所有新静态资源用 `?v=gap1` 避免缓存

## 五、风险与对策

| 风险 | 对策 |
|---|---|
| 新增 echarts CDN 失败 | 原 CSS 视图保留 + 优先级检测 typeof echarts |
| 顶栏入口过多 | 新增入口折叠到「更多」下拉 |
| 新建页面路由冲突 | 使用明确的 `.html` 文件名避免 |
| localStorage 污染 | 统一 `wm_*` 前缀 |
| 异议数据持久化 | localStorage key `wm_disputes` |
