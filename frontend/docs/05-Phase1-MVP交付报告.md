## 三、核心用户工作流（已跑通）

### 🎯 典型工作流 1：国企CFO 融资申请全链路

```
张总登录（U003 国企CFO）
    ↓
进入「我的工作台」(看到5个角色化KPI、待办、在办单据、快速操作)
    ↓
点击「快速操作 → 发起融资」
    ↓
5步向导（ticket-new.html?type=financing）
    Step 1 融资需求（金额200000/期限5-10年/用途/紧急度）
    Step 2 担保与偏好（集团信用/产品偏好/绿色/利率）
    Step 3 AI智能诊断（1.5秒 AI处理模拟）
    Step 4 选择方案A（公司债30亿+绿色债20亿，综合成本3.68%）
    Step 5 选择审批人（U006赵董事长）
    ↓
提交 → 生成单据 TK-FIN-xxx
    ↓
跳转「单据详情页」(状态：审批中)
    · 单据头+业务信息
    · 5节点审批流可视化
    · 处理时间线
    · 关联智能体（融资支持）
    · 相关人员
```

### 🎯 典型工作流 2：董事长审批

```
角色切换（下拉菜单 → U006 赵董事长）
    ↓
页面自动 reload，Topbar变为"赵董事长"
    ↓
进入「待办中心」(看到3条待办，含1条待审批)
    ↓
点击待审批项 → 单据详情
    ↓
右上角按钮变为「✓ 审批通过」「✗ 驳回」
    ↓
点击「审批通过」→ Modal 输入意见「同意」
    ↓
单据状态：审批中 → 已批准
    ↓
时间线新增记录
    ↓
发起人张总收到审批通过消息
```

## 五、单据工作流状态机

```
           ┌─────────┐
           │  Draft  │ (起草 · 仅创建人可见可改)
           └────┬────┘
                │ 提交
           ┌────▼─────┐
           │Submitted │ (已提交)
           └────┬─────┘
                │
           ┌────▼─────┐
        ┌──│Reviewing │ (审批中 · 当前审批人介入)
  驳回  │  └────┬─────┘
        │       │ 通过
   ┌────▼──┐    │
   │Rejected│ ┌─▼──────┐
   │       │ │Approved│ (已批准)
   │(可重交)│ └────┬───┘
   └───┬───┘      │ 开始执行
       │     ┌────▼────┐
       └────▶│Executing│ (执行中)
             └────┬────┘
                  │ 完成
             ┌────▼────┐
             │Completed│ (已完成)
             └────┬────┘
                  │ 归档
             ┌────▼────┐
             │Archived │ (已归档)
             └─────────┘
```

每次状态变更自动：
- 记录到 timeline（带时间/操作人/意见）
- 推送消息给相关人员
- 更新"我的待办"列表

本期交付已完整实现这个闭环，不仅 CFO 可以走完融资流程，7类角色的任何业务工作流都可以通过相同的"统一单据"体系完成。
