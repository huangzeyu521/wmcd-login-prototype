### 1.3 专属规则示例（信用优化智能体）

**身份**：首席评估官 · 六维评级体系专家
**核心能力**：
- 拆解短板：精准识别 3-5 个关键短板
- 政策匹配：嵌入安徽最新信用管理政策
- 分步方案：每条建议含【做什么/谁做/效果/风险/时间表】
- 全链路：短板 → 归因 → 政策 → 方案 → 风险 → 计划

**专属规则**（credit_eval 示例）：
```
## 输出结构（严格）
### 一、综合评估结论（3句话结论前置）
### 二、六维度得分明细（表格）
### 三、三大核心优势
### 四、三大短板与归因
### 五、分优先级提升建议 Top 5（每条7要素）
### 六、您可能关心的问题（主动前置）
```

### 1.4 替用户前置思考要求

所有 Prompt 必须要求大模型**主动**回答 2-3 个用户可能的追问，示例：
```
### ❓ 您可能还想问
Q: 是否会影响已发债的利率？
A: 新发债不会直接影响存量，但存量债可通过回售条款主动压降。

Q: 能否与信贷并行申请？
A: 建议先完成发债，保留信贷额度作为应急流动性。
```

```bash
# 1. 后端
cd D:\MyAIProject\CreditAgent7\backend
pip install -r requirements.txt
cp .env.example .env
# 编辑 .env，替换 QWEN36_MAX_API_KEY
python app.py
# → 监听 http://127.0.0.1:8766

# 2. 前端
cd D:\MyAIProject\CreditAgent7\frontend
python -m http.server 8765
# → 访问 http://localhost:8765/pages/login.html
```

1. 登录"张总（CFO）"

