# 《大模型调用思考过程可视化解决方案》

> 文档编号：WMCD-AI-THINKING-V1.0 · 2026-04
> 核心宗旨：**彻底解决用户"傻等"焦虑，让 AI 调用全程可见、可控、可撤销**

## 一、问题定义

### 1.1 用户体验三大痛点

| 痛点 | 场景 | 用户感受 |
|------|------|---------|
| 空白等待 | 点击"AI 分析"后 30 秒空白屏 | "卡死了？" |
| 黑盒推理 | 不知道 AI 是怎么推理的 | "真靠谱吗？" |
| 异常无感 | 网络超时、限流无任何提示 | "怎么没反应？" |

### 1.2 体验设计目标

> **Zero Dead Time · 零死寂时间**：用户点击后的每一秒都有**可视反馈**

## 二、技术架构：SSE 流式 + 三层事件

### 2.1 全链路架构
```
[用户点击 AI 按钮]
        ↓
[前端 ai-client.js · fetch + SSE]
        ↓
[Flask /api/ai/agent/xxx/stream]
        ↓
[Qwen Client · qwen3-max-preview + enable_thinking]
        ↓
[SSE 流：step / thinking / content / done / error]
        ↓
[前端 AI 面板 · 进度条 + 思考链 + 结果]
```

### 2.2 SSE 事件规范

| 事件 | 用途 | 示例 data |
|------|------|-----------|
| `step` | 6 步流程的阶段更新 | `{step: 3, total: 6, name: "深度推理", status: "running", msg: "..."}` |
| `thinking` | 大模型思考链片段（流式） | `"我需要先分析企业信用等级..."` |
| `content` | 最终结果片段（流式） | `"### 综合判断\n\n皖能集团..."` |
| `token` | Token 使用统计 | `{prompt_tokens: 450, completion_tokens: 1200}` |
| `done` | 全部完成 | `{total_chars: 2800, mock: false}` |
| `error` | 异常 | `{code: "timeout", message: "超时请重试，输入已保留"}` |

### 2.3 六步标准化流程

所有 AI 调用统一经历 6 个可视化步骤：

| 步骤 | 名称 | 含义 |
|------|------|------|
| ① | 数据解析 | 解析用户输入与业务上下文 |
| ② | 规则匹配 | 匹配国资专属知识库与属地政策 |
| ③ | 深度推理 | 大模型进行深度推理（含思考链） |
| ④ | 方案生成 | 组织专业答复（流式输出） |
| ⑤ | 合规校验 | 内容合规审查（敏感词/政策红线） |
| ⑥ | 结果输出 | 生成完成 |

## 三、分场景展示规则

### 场景 A：长耗时深度推理（如信用评估、融资方案）
- **进度条**：6步分步进度条，当前步骤脉动效果
- **左侧思考链**：实时流式滚动显示
- **右侧结果**：Markdown 格式化渲染，光标闪烁提示流式中

### 场景 B：常规交互（AI 助手对话）
- **抽屉模式**：右下角悬浮 FAB → 展开聊天抽屉
- **紧凑思考链**：思考链折叠显示（60px 高度，可展开）
- **气泡对话**：用户气泡（红底）+ AI 气泡（白底带边框）

### 场景 C：异常处理
| 异常类型 | 展示 |
|---------|------|
| 超时 | "网络连接大模型服务超时，请稍后重试。您的输入已保留。" + 🔄 重试按钮 |
| 限流 (429) | "当前调用人数较多，正在为您重试…"（自动退避重试） |
| 认证失败 (401) | "大模型服务认证失败，请检查 API Key 是否正确。"（管理员操作提示） |
| 内容过滤 | 合规校验步骤显示 `warning`，标注敏感点位 |
| 用户主动取消 | "已取消" 提示，支持重新启动 |

## 四、交互优化规则

### 4.1 所有场景通用
- ✅ **取消按钮**：全程可用，点击即终止
- ✅ **重试按钮**：异常后一键重试（输入已保留）
- ✅ **复制按钮**：结果完成后出现
- ✅ **Token 统计**：实时显示 Token 消耗与预估成本

### 4.2 高级用户 vs 普通用户
- **专业用户**：思考链默认展开，支持查看详细推理链
- **普通用户**：思考链默认折叠（工具栏"折叠/展开"按钮切换）

### 4.3 加载状态流畅性
- ✅ **无跳变**：步骤切换使用过渡动画
- ✅ **无卡顿**：SSE 事件即时处理，无批处理延迟
- ✅ **光标闪烁**：流式文本末尾光标指示"正在生成"

## 六、技术实现关键代码

### 6.1 前端 SSE 消费
```javascript
const resp = await fetch(url, {
    method: 'POST',
    headers: {'Content-Type': 'application/json'},
    body: JSON.stringify(body),
    signal: controller.signal,  // 支持取消
});
const reader = resp.body.getReader();
const decoder = new TextDecoder('utf-8');
let buf = '';

while (true) {
    const {done, value} = await reader.read();
    if (done) break;
    buf += decoder.decode(value, {stream: true});

    // 解析 SSE 帧
    let idx;
    while ((idx = buf.indexOf('\n\n')) >= 0) {
        const frame = buf.slice(0, idx);
        buf = buf.slice(idx + 2);
        // 分离 event: 和 data:
        const {event, data} = parseSSEFrame(frame);
        onEvent(event, data);
    }
}
```

### 6.2 后端 SSE 事件生成
```python
def _sse(event: str, data) -> str:
    body = json.dumps(data, ensure_ascii=False)
    return f'event: {event}\ndata: {body}\n\n'

@app.post('/api/ai/agent/<agent>/stream')
def ai_agent_stream(agent):
    def gen():
        for evt in qwen_client.chat_stream(...):
            yield _sse(evt['event'], evt['data'])
    return Response(gen(), mimetype='text/event-stream',
                    headers={'X-Accel-Buffering': 'no'})  # Nginx 禁止缓冲
```
