## 一、文档研读完成简报

### 1.1 核心定位
- **平台定位**：以新型人工智能基础设施为底座，覆盖全省国资国企的"信用+"智能平台
- **战略锚点**：服务安徽"三地一区"建设 + 十一大产业体系 + "十五五"改革
- **四方协同**：安徽省国资委 + 远东信用产业集团 + 中国科学技术大学 + 各级国资国企
- **核心引擎**：国资国企信用垂直领域大模型（计划以 qwen3-max-preview 为能力底座）

### 1.2 核心功能模块（三层架构）
| 层级 | 模块 | 包含 |
|------|------|------|
| L1 基础能力层 | 算力/数据/模型/安全/Multi-Agent | 8 个核心模块 F001-F008 |
| L2 核心业务层 | 信用标准/评估/监测/报告/共享 | 6 个核心模块 F101-F106 |
| L3 智能应用层 | 七大智能体 + 监管驾驶舱 | 8 个核心应用 F201-F208 |

### 1.3 大模型能力适配方向
本清单识别出 **38 个需大模型深度介入**的功能点，分布：
- 信用优化智能体：6 项
- 融资支持智能体：5 项

- 供应链管理智能体：4 项
- 风险防控智能体：5 项
- 公共能力：3 项

### 1.4 用户体验核心痛点（需思考过程可视化解决）
| 痛点 | 场景 | 解决方向 |
|------|------|---------|
| 长耗时"傻等" | 信用全景分析（≥30秒）、投资尽调（≥2分钟） | 分步进度条 + 实时思考日志 |
| 推理过程黑盒 | AI评级依据不清、融资方案凭什么推荐 | 左侧思考链 + 右侧最终结果 |
| 网络波动焦虑 | SSE 断开、Token 超限、限流 | 友好提示 + 自动重试 + 保留输入 |
| 结果不够专业 | 通用模型缺国资专属知识 | 专属 Prompt + 属地政策嵌入 |

## 三、《需接入 qwen3-max-preview API 的功能清单 V2.0》

**汇总：38 个 AI 核心功能点**，全部通过 SSE 流式 API 实现思考过程可视化。

### 3.1 接入优先级分层

| 层级 | 功能点数 | 优先级 | 说明 |
|------|---------|--------|------|
| P0 核心 | 12 | 最高 | 覆盖所有长耗时深度推理场景 |
| P1 重要 | 18 | 高 | 覆盖常规交互与短耗时场景 |
| P2 可选 | 8 | 中 | 非关键路径，后续迭代 |

### 3.2 API 调用触发条件与输入输出规范

每个功能点均遵循以下标准结构：

```
入参：
- agent_id: 'credit'|'financing'|'investment'|'assets'|'ma'|'supply'|'risk'
- function: 功能点ID（如 'credit_eval'）
- context: 业务上下文对象（企业信息、金额等）
- user_role: 当前用户角色
- enable_thinking: bool 是否启用思考链输出

出参（SSE 流式）：
- event: 'step' 步骤进度更新
- event: 'thinking' 思考过程（流式）
- event: 'content' 最终答复（流式）
- event: 'done' 完成
- event: 'error' 异常
```

### 3.3 合规约束

- ✅ 国资敏感数据脱敏后才传给大模型
- ✅ 所有调用全程审计日志
- ✅ 输出内容经合规过滤（32/42号令 红线关键词）
- ✅ 支持人工介入/人工终审
- ✅ 生成内容水印/可追溯
