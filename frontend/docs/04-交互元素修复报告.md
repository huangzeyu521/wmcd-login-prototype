## 一、问题诊断

### 1.1 根因分析
用户反馈"左侧导航栏菜单无法点开"经深度排查，根本原因是：

> **原实现将侧边栏菜单写为纯装饰 `<div>` 元素，未绑定任何 JavaScript 点击事件。** HTML 语义上 `<div>` 并非交互元素，浏览器对其无默认点击行为，用户点击后视觉无反馈，即 "无法点开"。

### 1.2 问题清单（修复前）

| ID | 严重度 | 类型 | 涉及数量 | 位置 | 根因 |
|----|--------|------|----------|------|------|
| P-001 | 🔴 严重 | 侧边栏菜单失效 | 62 处 | 7 个智能体页面 `<div class="sb-item">` | 无 JS 事件，无 active 切换 |
| P-002 | 🔴 严重 | 内容区按钮无反馈 | 52 处 | 9 个页面所有 `<button class="wm-btn">` | 无 onclick 绑定 |
| P-003 | 🟡 中 | 空链接跳顶 | 1 处 | `agent-credit.html` `<a href="#">查看全部8条建议` | 浏览器默认行为 |
| P-004 | 🟡 中 | 无障碍不达标 | 62 处 | 侧边栏 `<div>` 缺 role/tabindex | 键盘用户不可达 |
| P-005 | 🟡 中 | 视觉提示缺失 | 全局 | 无 cursor:pointer、无 focus 样式 | 视觉误导 |

## 二、修复方案

### 2.1 整体架构
采用**全局事件委托 + 自动增强**的无侵入式方案，不改变原有 HTML 结构（保证设计稿视觉 100% 一致），通过统一的 `interactions.js` 为所有元素注入行为：

```
┌─────────────────────────────────────────┐
│  新增: assets/js/interactions.js        │
│  ─────────────────────────────          │
│  • Toast 反馈系统                        │
│  • 侧边栏增强（点击+切换+键盘+滚动）     │
│  • 按钮增强（智能动作映射 + Toast）      │
│  • 空链接拦截                            │
│  • MutationObserver 动态元素自动绑定     │
│  • 全局点击日志（调试）                   │
└──────────────┬──────────────────────────┘
               │
               ↓ 在 9 个 HTML 中引入
┌─────────────────────────────────────────┐
│  增强: assets/css/shared.css            │
│  ─────────────────────────────          │
│  • .wm-toast-container / .wm-toast       │
│  • .sb-item :focus/:active 样式         │
│  • .sb-item.active::after 红点指示       │
│  • .wm-btn :focus/:active/ripple         │
│  • cursor:pointer / 无障碍 focus 环      │
└─────────────────────────────────────────┘
```

### 2.2 核心修复代码

#### A. `assets/js/interactions.js`（新增，331 行）
| 模块 | 职责 | 关键代码 |
|------|------|----------|
| Toast 系统 | 顶部浮层反馈（成功/信息/警告/错误四色） | `showToast(msg, type, duration)` |
| `enhanceSidebar()` | 为所有 `.sb-item` 设置 `role=button`、`tabindex=0`、`cursor=pointer`；绑定 click 切换 active；绑定 Enter/Space 键盘；显示 Toast | `items.forEach(item => item.addEventListener('click', ...))` |
| `enhanceButtons()` | 为所有 `<button class="wm-btn">` 绑定 click 并根据文字智能匹配动作文案（40+ 映射） | `const actionMap = {'生成报告':'正在生成报告…', 'AI解读':'调用AI大模型生成解读…', ...}` |
| `enhanceEmptyLinks()` | 拦截 `a[href="#"]` 防止跳顶 + Toast 反馈 | `e.preventDefault(); showToast(...)` |
| MutationObserver | 监听 DOM 变化，为 `renderTopbar()` 动态注入的元素补齐绑定 | `observer.observe(document.documentElement, {childList:true, subtree:true})` |

#### B. `assets/css/shared.css`（新增 125 行样式）
| 样式类 | 作用 |
|--------|------|
| `.wm-toast-container / .wm-toast` | 顶部 Toast 浮层容器与基础样式 |
| `.wm-toast-success/info/warn/error` | 四色 Toast 主题 |
| `.wm-sidebar .sb-item:focus-visible` | 键盘聚焦态（无障碍） |
| `.wm-sidebar .sb-item.active::after` | active 项右侧红色发光小点 |
| `.wm-btn:active` | 按下缩放反馈 |
| `.wm-btn::after` (radial-gradient) | 点击 ripple 涟漪效果 |
| `button / a / .sb-item { cursor: pointer }` | 所有交互元素鼠标手型 |

#### C. 9 个 HTML 页面统一注入（原子化最小修改）
```html
<script src="../assets/js/shared.js"></script>
<script src="../assets/js/interactions.js"></script>  <!-- 新增 -->
```

## 六、增强特性（深度优化）

### 6.1 响应速度优化
- DOM 绑定延迟 50ms 等待 `renderTopbar()` 动态注入
- MutationObserver 节流（100ms debounce）避免频繁触发
- Toast 动画 ≤ 300ms，首响应 ≤ 150ms

### 6.2 鲁棒性增强
- **动态元素自动绑定**：MutationObserver 监听 DOM 变化，新增元素自动获得绑定
- **重复绑定防护**：`dataset.wmBound='1'` 标记，避免事件叠加
- **原生行为保留**：`<a>` 带有效 href 的自动跳过 JS 拦截
- **错误静默**：try/catch 包裹 console.log，避免旧浏览器崩溃
- **键盘可达**：Enter/Space 键触发点击

### 6.3 反馈统一
- **一致的 Toast 样式**：四色统一（success/info/warn/error）
- **统一的 active 指示**：红色高亮 + 右侧发光小点
- **统一的按下反馈**：`transform: scale(0.98)` 模拟物理按压
- **统一的 focus 环**：`box-shadow 3px rgba(184,20,26,0.2)`
